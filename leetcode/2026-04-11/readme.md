## 258 

### 题目

给定一个非负整数 `num`，反复将各个位上的数字相加，直到结果为一位数。返回这个结果。

**示例 1:**

```
输入: num = 38
输出: 2 
解释: 各位相加的过程为：
38 --> 3 + 8 --> 11
11 --> 1 + 1 --> 2
由于 2 是一位数，所以返回 2。
```

**示例 2:**

```
输入: num = 0
输出: 0
```

**提示**

- `0 <= num <= 231 - 1`

**进阶：**你可以不使用循环或者递归，在 `O(1)` 时间复杂度内解决这个问题吗？



### 理解

看LeetCode官方用了很数学系的数学来解释这件事情，评论区用了比较形象的分堆之类的实际意义来解释，这两个角度都不太符合自己的看法，下面是用比较“**口语数学**”的方式来解释这件事情

首先我们需要取模也是有分配律的（因为不是做数论的，我先前还不知道有这件事情）

比如一个数
$$
123 \% 2 = 1
$$
可以等价于
$$
(100+20+3)\%2 =[ 100\%2+20\%2 + 3\%2]\%2 =[0+0+1]\%2=1
$$
这样我们就可以归纳成
$$
\begin{align}
(\cdots+100a_1+10a_2+1a_3) \% m &=[\cdots+100a_1\%m+10a_2 \%m+1a_3\%m]\%m\\
\end{align}
$$
根据评论区的思路[O(1)解法的个人理解](https://leetcode.cn/problems/add-digits/solutions/15652/java-o1jie-fa-de-ge-ren-li-jie-by-liveforexperienc/)，我们把他拆分成(999+1)、(99+1)、(9+1)这样的，

为什么可以这样呢？

**继续看$123$这个例子**
$$
123 = 1\times(99+1)+2\times(9+1)+3 = (1\times99+2\times9)+1+2+3
$$
**如果我们把$(1\times99+2\times9)$这个东西去掉，保留$1+2+3$**

还是根据[前人的思路]([LeetCode—各位相加（循环法+模九法）-CSDN博客](https://blog.csdn.net/weixin_41541562/article/details/106635899))，$(1\times99+2\times9)$模上9不就清空了吗，$1+2+3$模9不就被保留了吗，什么操作能做到这件把他们拆成99\*多少+9\*+一大堆数字这样的事情呢？前文我们介绍了公式1，刚刚好了

所以只需要
$$
123\%9
$$
即可





**但是还有一个问题**，就是如果是9的倍数不就变成0了吗
$$
81\%9=0
$$
。。但是我们知道预期的结果是$8+1=9$啊

这时候就可以-1，再加1完成修补

就是
$$
(81-1) \% 9 + 1 = 8+1 = 9 
$$
Over



### 代码

```c++
class Solution {
public:
    int addDigits(int num) { return (num - 1) % 9 + 1; }
};
```
